### 브라우저의 렌더링 원리?

CRP <br />

블링크, 웹킷 같은 렌더링 엔진을 통해<br />
html을 파싱후 dom트리를 구축,<br />
그후 css를 파싱해 cssom트리를 구축,<br />
다음 단계로 js를 실행하며 dom과 cssom을 조합해 노드가 가지는 위치와 크기를 계산하는<br />
layout단계를 거치고, 계산된 위차와 크기를 기반으로 화면에 Paint해 자원을 픽셀화해 화면에 나타냅니다.<br />

### 주소창에 구글을 입력하면 일어나는 일?

브라우저는 사용자가 입력한 URL중 도메인 이름 부분을 DNS서버에서 검색합니다.<br />
DNS서버에서는 해당 도메인 이름에 해당하는 ip주소를 찾아 사용자가 입력한 URL정보와 함께 브라우저에 전달하며,<br />
브라우저는 HTTP프로토콜을 사용해 요청메시지를 만들어 서버로 전달합니다.<br />
서버는 RES메시지를 생성해 브라우저에 응답 데이터로 전송합니다.<br />
브라우저는 받은 데이터를 바탕으로 화면을 렌더링 하게 됩니다.<br />

- dns: www.naver.com과 같은 사람이 읽을수 있는 도메인이름 <br />

- url: 통합 자원 지시자로서 인터넷의 리소스를 가리키는 표준 명칭 <br />

- http: tcp기반으로 클라이언트와 서버사이 이루어지는 요청/응답 프로토콜. 요청이 있으면 응답이 있어야한다. <br />

- 프로토콜: 서버와 클라언트 통신을 위한 약속을 정의해 둔것. <br />

- tcp: 두개의 호스트를 연결하고 데이터 스트림을 교환하게 해주는 네트워크 프로토콜. 데이터를 어떻게 보내고 제어할것인지 정의할것지를 정합니다. <br />

### 호이스팅

변수 및 함수 선언문이 스코프 내 최상단으로 끌어올려지는 현상을 말한다.<br />

- 인터프리팅: 컴파일과 다르게 소스코드를 한번에 읽어서 번역하지 않고, 런타임 상태에서 소스코드를 한줄한줄 번역하면서 프로그램을 구동하는 방식. 중간 코드로 번역되며 <br />
또 다시 다른 프로그램을 통해 기계어로 번역되어 실행된다. <br />


### 클로져

정말 많은 정의들이 있지만, 클로저는 어떤함수(outter) 내부에 선언된 함수(inner)가 outter의 지역변수를 참조하는 것이<br />
outter가 종료된 이후에도 유지되는 현상이라고 말씀드리고 싶습니다. 이것은 렉시컬 스코프의 영향 <br />

- 렉시컬스코프: 함수를 어디서 호출하는지가 아닌 어디에 선언하였는지에 따라 상위 스코프가 결정되는 것을 말한다.<br />

### URL ?

서버에 자원을 요청하기 위해 입력하는 영문주소<br />

### HTTP 1 vs HTTP 2

1은 기본적으로 커넥션 하나당 하나의 요청과 응답을 처리한다.<br />
반면 2는 여러 리소르를 동시전송이 가능하므로 1에 비해 로드 속도가 빠르다.<br />

### HTTP와 HTTPS차이

https는 http에 데이터 암호화가 추가된 프로토콜이다. <br />
http와 다르게 포트 443을 사용하며, 브라우저를 통해 입력된 프라이빗한 정보들을 암호화를 지원하며,<br />
입력된 정보는 응답하는 서버만 확인가능하다. 또한 검증된 사이트라는 마커로의 의미도 같는다.<br />

### 쿠키와 세션같은 저장소를 쓰는이유?

http는 항시 연결되어있는 것이아니다. 요청이 있을때만 응답을 주는 비연결성이 있고,<br />
다른페이지는 다른 탭을 통해 다시 접속시 그정보가 유지되어야 하지만 계속 요청하는 작용은<br />
비현실적으므로 어딘가에 저장할 필요가있다.<br />

### 쿠키와 세션에 대해 설명해주세요.

쿠키는 클라이언트에 저장되는 작은 데이터로 4kb의 300개 정도 저장이 가능하다.<br />
서버에서 발급한 쿠키를 클라이언트에서 저장하며 요청시 헤더에 넣어 전송하는 방식이다.<br />
유효시간을 지정할수 있어 브라우저가 닫히더라도 유지가능하다.<br />
(아이디 기억이나, 이창보지않음에 사용)<br />
세션 역시 쿠키기반으로 서버측에서 세션id를 부여하여 브라우저 종료시까지 인증 상태를 유지하며<br />
클라에서 세션id를 쿠키에 저장해 요청시 저장된 세션id로 상태를 유지한다.<br />
서버측에서 관리하기 떄문에 보안상 이점이 있으나 요청시마다 서버처리가 필요해 과부하의 위험이 있다.<br />

### 로컬 스토리지, 세션 스토리지

서버에 불필요한 데이터 저장 없이 웹 스토리지로서 저장하며, 5mb로 비교적 용량이 크다<br />
로컬스토리지는 영구적이면 세션스토리지는 브라우저 종료시까지 유지된다.<br />

동기라고 함은 요청을 보낸후 응답을 받아야 다음 동작을 실행하며 반면, 비동기라고 함은<br />
요청을 보낸 후 응답과 관계없이 다음 동작을 수행하는 것을 말한다.<br />

### Promise와 async await에 대해 설명해 달라

두가지 모두 promise가 처리될때까지 결과를 반환하지 않고 대기 하는 비동기처리 문법 입니다.<br />
먼저 promise는 조건에 부합하면 resolve를 호출하고, 아닐 시 reject를 호출합니다.<br />
resolve한 값에 대해서 then을 통해 값을 받고 reject의 값에 대해서는 catch를 통해 에러 상태를 반환합니다.<br />

async await은 call back 지옥과 then의 꼬리물기를 해결하기 위한 대안책으로 try-catch를 통해<br />
에러핸들링과 async await라는 규칙을 줌으로 가독성과 코드흐름을 이해하기 쉽게 도와줍니다.<br />

### CORS라는 무엇이고 CORS에러 실무해결 방법?

CORS란 한 도메인 또는 origin의 웹페이지가 다른 도메인을 가진 리소스에 액세스 할 수 있게하는 보안 메커니즘.ㅍ<br />
결국 cors에러가 발생하는 이유는 접근 타겟이 되는 도메인이 리소스 접근 허용을 하지않는 것입니다.<br />

해결했던 경험은? 실무에서 cors세팅은 기본적으로 클라이언트에서는 리퀘스트 해더에 크리덴셜 세팅이나 프록시를<br />
통한 설정을 통해 해결을 하는데 다방에서 백오피스 설계를 하여 api test중 cors 오류가 발생한적 이있다.<br />
사실 클라이언트에서 필수적인 설정은 해놓았긴때문에 이유를 찾기 힘들어 서버개발자화 함께 상황을 지켜보니<br />
서버에서 access-control-allow-origin 도메인을 오타를 냈다고 하였다. 서로 웃으면서 해프닝으로 넘어간<br />
기억이 있습니다.<br />

### 타입스크립트란?

타입을 명시하지 않는 유연한 자바스크립트에 정적 타입을 지원해 컴파일 단계에서 오류를 발견하고<br />

### SPA, CSR, SSR

- SPA : 하나의 HTML 파일을 기반으로 자바스크립트를 이용해 동적으로 화면의 컨텐츠를 바꾸는 방식의 웹 어플리케이션.<br />

- CSR : CSR에선 브라우저가 서버에 HTML과 JS 파일을 요청한 후 로드되면 사용자의 상호작용에 따라 JS를 이용해서 동적으로 <br />렌더링을 시킨다. 초기로딩이 느리고 js를 통해 화면을 구성하기에 검색엔진 최적화가 잘안된다.<br />

- ssr : SSR에선 브라우저가 페이지를 요청할 때마다 해당 페이지에 관련된 HTML, CSS, JS 파일 및 데이터를 받아와서 <br />렌더링을 시킨다. 초기 로딩이 빠르며 검색엔진 최적화가 된다. 배번 서버요청과 페이지이동시 새로고침 현상으로 사용자 경험이
좋지 않다.<br />

### 리액트에서 불변성을 지켜야하는 이유?

불변성은 어떤 값을 직접적으로 변경하지 않고 새로운 값을 만들어내는 것인데,<br />
사실 실무에서 스프레드혹은 immer을 이용해 불변을 지켜주는 이유는,<br />
리엑트 상태 변화를 감지하여 화면을 효과적으로 변경하는 라이브러입니다.<br />
따라서 변경된상태를 감지 하지 못하면 이를 수행할 수 없습니다.<br />
따라서 이전 상태를 유지하고 새로운 값을 비교함으로 화면변화를 나타냅니다.<br />

요청이 들어와도 데이터의 변화는 없고 미리 렌더링해두어도 괜찮다 그러면 getStaticProps 를 사용하자! <br />

### JWT

JSON 객체를 사용해서 토큰 자체에 정보들을 저장하고 있는 Web Token 으로 가벼운 인증으로 사용이 쉽다. 일반적으로 <br />클라이언트와 서버, 서비스와 서비스 사이 통신을 할 때 권한 인가(Authorization)을 위해서 사용된다.

### 일반함수

자바스크립트에서 모든 함수는 실행될 때마다 함수 내부에 this라는 객체가 추가된다. <br>
함수 실행시에는 전역(window) 객체를 가리킨다.<br>
메소드 실행시에는 메소드를 소유하고 있는 객체를 가리킨다.<br>
생성자 실행시에는 새롭게 만들어진 객체를 가리킨다.<br>
일반 함수는 함수를 선언할 때 this에 바인딩할 객체가 정적으로 결정되는 것이 아니고,<br>
함수를 호출할 때 함수가 어떻게 호출되었는지에 따라 this에 바인딩할 객체가 동적으로 결정된다.<br>
일반 함수는 생성자 함수로 사용할 수 있다.<br />

### 화살표함수

화살표 함수는 함수를 선언할 때 this에 바인딩할 객체가 정적으로 결정된다.<br>
화살표 함수의 this 언제나 상위 스코프의 this를 가리킨다.(Lexical this)<br>
또한 call, apply, bind 메소드를 사용하여 this를 변경할 수 없다.<br>
화살표 함수는 생성자 함수로 사용할 수 없다. prototype 프로퍼티를 가지고 있지 않기 때문이다.<br>
